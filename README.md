# MediaWiki 分类交集工具
**查找同时属于多个分类的页面**

## 功能特性
- **站点连接**：支持连接任意 MediaWiki 站点的 API
- **分类管理**：浏览、搜索分类，支持屏蔽空分类（页面数为0的分类）
- **交集计算**：计算多个分类的页面交集
- **结果排序**：支持按页面标题、最后修改时间、页面ID排序
- **Excel 导出**：支持导出结果到 Excel，可选择包含超链接
- **国际化**：支持中英文界面切换
- **性能优化**：大数据集使用 Web Worker 计算，避免 UI 阻塞

## 使用说明
1. **连接站点**： 在 “MediaWiki 站点连接” 部分输入目标 MediaWiki 站点的 api.php 文件的完整 URL `(例如：https://zh.wikipedia.org/w/api.php)`，然后点击 “连接并加载分类” 按钮。
2. **选择分类**： 连接成功后，可在 “分类选择” 部分浏览或搜索分类。点击分类名称将其添加到 “已选分类” 列表。您可以选择多个分类。
3. **查找交集**： 选择完所有目标分类后，点击 “查找所有已选分类中的页面” 按钮。工具将开始获取并计算这些分类下的页面交集。
4. **查看结果**： 处理完成后，“交集结果” 部分将显示同时属于所有已选分类的页面列表。点击表头可对结果进行排序。如果结果过多，将提供分页导航。
5. **导出数据**： 可将结果导出为 Excel 文件，支持选择是否包含超链接。

## 部署与使用
- **本地使用**：直接打开 `index.html` 即可
- **服务器部署**：将所有文件上传到服务器即可
- **[在线示例](http://long.club/wiki/)**

## 技术说明
- **前端框架**：原生 JavaScript + Tailwind CSS
- **API 交互**：直接调用 MediaWiki API
- **性能优化**：Web Worker 处理大数据集
- **导出功能**：使用 SheetJS 库导出 Excel

## 注意事项
- **CORS 限制**：本工具未解决跨域资源共享(CORS)问题。如果 MediaWiki 站点的 API 没有启用 CORS，可能无法正常访问。

## 推荐扩展
对于 MediaWiki 站长，推荐安装以下扩展以在站内实现类似功能：
- **[DynamicPageList4](https://www.mediawiki.org/wiki/Extension:DynamicPageList4)** - 功能强大的页面列表生成器，支持分类交集
- **[DynamicPageList](https://www.mediawiki.org/wiki/Extension:DynamicPageList)** - 经典的分类交集扩展

## 开发背景
该工具最初为书籍/小说收集的 wiki 编写，用于处理多分类内容过滤需求。
